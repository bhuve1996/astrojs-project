## ğŸ“¦ Steps to Move Project

### Option 1: Move to New Folder/Repository

1. **Copy the entire `website-cloner` folder:**
   ```bash
   # From parent directory
   cp -r website-cloner /path/to/new/location/
   # Or
   mv website-cloner /path/to/new/location/
   ```

2. **Or create a new git repository:**
   ```bash
   cd website-cloner
   git init
   git add .
   git commit -m "Initial commit: Website cloner with metadata extraction"
   git remote add origin <your-new-repo-url>
   git push -u origin main
   ```

### Option 2: Copy to Another Repository

1. **If you have an existing repository:**
   ```bash
   # Copy all files except node_modules
   rsync -av --exclude 'node_modules' --exclude '.astro' --exclude 'dist' \
     website-cloner/ /path/to/your/repo/
   ```

2. **Then install dependencies:**
   ```bash
   cd /path/to/your/repo
   npm install
   ```

### Option 3: Create a Clean Copy (Recommended)

1. **Copy only essential files:**
   ```bash
   # Create new directory
   mkdir -p /path/to/new/project
   
   # Copy essential files
   cp -r website-cloner/src /path/to/new/project/
   cp -r website-cloner/scripts /path/to/new/project/
   cp website-cloner/package.json /path/to/new/project/
   cp website-cloner/package-lock.json /path/to/new/project/
   cp website-cloner/astro.config.mjs /path/to/new/project/
   cp website-cloner/tsconfig.json /path/to/new/project/
   cp website-cloner/.gitignore /path/to/new/project/
   cp website-cloner/README.md /path/to/new/project/
   cp website-cloner/DOCUMENTATION.md /path/to/new/project/
   cp website-cloner/MOVE_PROJECT.md /path/to/new/project/
   
   # Create necessary directories
   mkdir -p /path/to/new/project/public
   mkdir -p /path/to/new/project/scraped-data
   ```

2. **Install dependencies:**
   ```bash
   cd /path/to/new/project
   npm install
   ```

## ğŸ†˜ Troubleshooting

### If `npm install` fails:
- Check Node.js version (should be 18+)
- Delete `node_modules/` and `package-lock.json`, then run `npm install` again

### If scraper fails:
- Check internet connection
- Verify the target website is accessible
- Check browser console for errors

### If build fails:
- Ensure `scraped-data/website-data.json` exists
- Run scraper first: `npm run scrape <url>`
- Check for TypeScript errors

### If preview shows errors:
- Verify `public/assets/` folder exists
- Check that assets were copied from `scraped-data/assets/`
- Run `npm run build` again
